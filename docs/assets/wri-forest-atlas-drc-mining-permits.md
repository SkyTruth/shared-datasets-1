## What this is

This asset is a 2026-06-29 extract of mining permit polygons from the WRI Forest Atlas of the Democratic Republic of the Congo ArcGIS REST service, layer 101 (`Mining permit`). The source metadata says the layer is produced by the Mining Cadastre of the Democratic Republic of the Congo (CAMI) and used by the CARPE-WRI project for the Forest Atlas of the DRC.

The asset slug intentionally names WRI Forest Atlas DRC as the source because other DRC mining-permit sources may be added separately.

## When to use it

- Use this as an internal reusable DRC mining-permit polygon layer for extractives, land-use, and forest-risk analysis.
- Use the FGB for analysis; full attributes are also available in the metadata sidecar keyed by `feature_id`.
- Use the PMTiles artifact for web-map preview and feature lookup only. PMTiles features contain `feature_id`, not the full source attributes.
- Do not treat this as a CAMI-direct authoritative release unless the upstream source and terms are separately confirmed.
- Do not make this asset public until source reuse terms are confirmed and public access is explicitly approved.

## Schema notes

Geometry is WGS84 MultiPolygon geometry. The source service advertises Web Mercator coordinates and supports paged ArcGIS REST queries; this release was downloaded in objectid batches and normalized to EPSG:4326. The source service reported 3,595 records, and 3,595 features were downloaded and published.

The 2026-06-29 release uses the source ArcGIS `objectid` field as `feature_id`. It was selected because the full extract contains 3,595 unique nonblank object IDs that match the feature ID rules. `num_permis` is preserved for search but is not suitable as a feature ID because it contains blank and duplicate values.

`geometry_hash` and `properties_hash` are SHA-256 content hashes published for duplicate detection and refresh-time identity checks. Hashes were recomputed from the final published FGB export and match the canonical metadata sidecar for all 3,595 features.

No autogenerated localized metadata sidecars are included in the first upload. The source fields include French labels and values; translation targets should be reviewed separately before generating localized sidecars.
