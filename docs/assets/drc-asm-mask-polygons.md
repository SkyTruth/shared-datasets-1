## What this is

This asset is a polygon release derived from the binary artisanal and
small-scale mining (ASM) masks in Zenodo record 15257800, the dataset
accompanying Pasanisi et al. in Remote Sensing 17(24), article 4057. The source
archive contains paired satellite image tiles and binary masks for ASM
segmentation in the Democratic Republic of the Congo.

The canonical shared-datasets release uses `dataset/ps/masks/*.tif` as the
geometry source because those PlanetScope-aligned masks have the 4.77 m source
resolution. Sentinel-1 masks are spatially corresponding, lower-resolution
companions in the source archive and were not used to derive polygon geometry.

## When to use it

- Use this as a citable polygon label layer for ASM footprint inspection,
  benchmarking, and model evaluation in DRC.
- Use the FlatGeobuf for analytical workflows and the metadata sidecar for
  feature-level lookup by `feature_id`.
- Use the PMTiles artifact for web-map display or catalog preview only; tiles
  intentionally carry only `feature_id`.
- Do not treat this as a complete inventory of all DRC ASM activity, a legal
  mining-cadastre source, or a current operational monitoring layer.
- Do not use the PMTiles artifact as the analytical source.

## Geometry Generation Pipeline

The 2026-07-01 release was generated locally under
`/private/tmp/shared-datasets-1/vector-assets/drc-asm-mask-polygons` from
Zenodo `dataset.zip` checksum `md5:8b218368e42a3d926bf4b42b553e49f5`.

The pipeline was:

1. Download `dataset.zip` from Zenodo record 15257800 and verify the MD5.
2. Unzip the archive and confirm exactly 782 PlanetScope masks under
   `dataset/ps/masks` and 782 Sentinel-1 masks under `dataset/s1/masks`.
3. Inspect the PlanetScope masks for georeferencing, bounds, pixel size, nodata,
   and unique values. All PlanetScope masks are 256 by 256 GeoTIFFs with binary
   values `0` and `1`; 15 masks contain no ASM pixels.
4. Polygonize value `1` pixels from each PlanetScope mask with GDAL, retaining
   a raw audit layer with `source_mask` and `mask_value`.
5. Dissolve touching or overlapping polygons across all masks to produce one
   footprint per contiguous ASM area. The source masks include overlapping
   tiles, so dissolution removes duplicate covered area.
6. Repair geometries, explode multipart results, project to EPSG:6933 for area
   calculations, and filter dissolved parts below 100 m2.
7. Reproject the cleaned polygons to EPSG:4326, assign generated decimal
   `feature_id` values from `geometry_hash` and `properties_hash`, and build
   release-oriented FGB, PMTiles, metadata sidecar, schema, and manifest
   artifacts.

GDAL 3.13.1 with PROJ data from `/usr/local/Cellar/proj/9.8.1/share/proj` was
used for `gdalinfo`, `ogr2ogr`, and polygonization. Tippecanoe v2.79.0 and
`pmtiles` were used for metadata-lookup PMTiles generation.

## Schema notes

Geometry is WGS84 multipolygon geometry. The source raster mask extent inspected
from the PlanetScope masks is `(23.654029, -11.517469) - (30.590572,
4.469603)`. The published cleaned polygon extent is `(23.659642, -11.517469) -
(30.590572, 4.466646)`.

The release contains 6,840 cleaned footprint features. Total cleaned area is
67,183,793.125 m2. Raw per-mask polygon area summed before dissolve was
75,528,915.791 m2. Dissolving overlapping/touching polygons and filtering
slivers below 100 m2 removed 8,345,122.665 m2 relative to that raw per-mask sum;
the explicit below-100 m2 filter removed 2,984 parts totaling 131,952.909 m2.

The PMTiles artifact is a lightweight metadata-lookup layer with zooms 2 through
12 present in the archive metadata and `maxzoom` 12 selected from the 4.77 m
source resolution. Its features carry only `feature_id`; all other attributes
must be read from the FGB or metadata sidecar.

No autogenerated localized metadata sidecars were requested for this first
upload because the asset has no human-readable feature names or labels.

## Feature Identity Decision

No source field uniquely identifies a dissolved ASM footprint. Source mask names
are tile-level provenance, not footprint identity, and dissolved footprints can
merge pixels across adjacent or overlapping masks. The release therefore uses
generated monotonic decimal `feature_id` values assigned from the release-model
content identity key `(geometry_hash, properties_hash)`.

| Candidate | Datatype | Distinct non-empty values | Empty values | Domination / skew | Top examples | Decision / concern |
|---|---|---:|---:|---|---|---|
| `source_mask` from raw polygonization | string | 782 mask filenames before dissolve | 0 in raw layer | Not footprint-level; one mask can produce many polygons and one cleaned polygon can merge across masks | `mask_0.tif`, `mask_1.tif`, `mask_10.tif` | Rejected; tile provenance only and not published in the cleaned release layer |
| `mask_value` | integer | 1 | 0 | Dominated by value `1` for every retained feature | `1` | Rejected; class value, not an identifier |
| `area_m2` | float | Not stable as an identifier | 0 | Floating area values can change with geometry repair or reprojection details | `100.353...`, `245.812...` | Rejected; measurement, not source identity |
| `(geometry_hash, properties_hash)` | string hashes | 6,840 identity keys | 0 | One key per published cleaned footprint | `sha256:...` pairs | Selected assignment key for generated monotonic decimal `feature_id` values |

## Known caveats

This is a label dataset derived from binary segmentation masks, not a complete
or current inventory of all DRC ASM sites. It inherits all spatial, temporal,
and survey-label limitations described by the source dataset and companion
paper.

The canonical polygons are mask-footprint products. Pixel stair-stepping is
expected, and the 100 m2 threshold removes very small dissolved parts. Use the
raw audit layer retained in the local work directory if the sliver threshold
needs future review, but do not treat raw per-mask polygons as the canonical
release because overlapping source masks duplicate area.

The source archive includes PlanetScope image tiles derived from Planet-NICFI
basemaps. This shared-datasets asset publishes polygons derived from the CC BY
4.0 masks, but users should still cite the Zenodo record and respect the source
archive's Planet NICFI lineage and license caveat.
