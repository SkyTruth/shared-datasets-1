## What this is

Large Scale International Boundaries (LSIB) is the U.S. Department of State
Office of the Geographer and Global Issues international boundary line dataset.
Version 11.4 was published on February 24, 2025. The source metadata describes
LSIB and generalized derivatives as the only international boundary lines
approved for U.S. Government use. The dataset reflects U.S. Government policy on
international boundary alignment, political recognition, and dispute status; it
does not necessarily represent de facto limits of control.

The source package includes a shapefile, a test GeoPackage, QGIS and ArcGIS
project files, and example style files. The source README says the shapefile is
still the authoritative LSIB format and the GeoPackage is included for testing,
so this release was built from the authoritative shapefile.

## When to use it

- Use this as a reusable global international-boundary reference layer when
  U.S. Government-recognized boundary alignment and political status are the
  intended basis.
- Use the FlatGeobuf file for analytical workflows.
- Use the PMTiles file for web-map display and catalog preview.
- Review the source guidance before cartographic products: LSIB styling depends
  on `RANK`, `STATUS`, and `LABEL`.
- Do not use this as a neutral statement of de facto control, cadastral
  boundary source, or substitute for specialist legal boundary analysis.
- Do not use the PMTiles artifact as the analytical source.

## Schema notes

Geometry is WGS84 multiline geometry. The source shapefile reports 395 line
features with extent `(-141.001858, -55.153672) - (141.022151, 70.092293)`.
The published FlatGeobuf promotes geometries to multiline for consistent
analytical handling and keeps the source fields plus `feature_id`,
`geometry_hash`, and `properties_hash`.

The source metadata describes the field named `ID` in the shapefile as
`LSIB_ID`: a UUID value defining a specific instance of a feature. It is unique
and nonblank in this release, but UUID hyphens violate the shared-datasets
`feature_id` rule (`^[A-Za-z0-9]{1,64}$`). The reviewed 2026-06-26 release
therefore assigns generated monotonic decimal `feature_id` values (`1`..`395`)
from the source `ID` field and records that source field as the assignment key.

The PMTiles artifact was built at maxzoom 12 from the 100-meter source accuracy
hint recorded in State Department metadata. PMTiles feature properties are
intentionally limited to `feature_id`; full source attributes are available in
the FlatGeobuf and metadata sidecar.

No autogenerated localized metadata sidecars were requested for this first
upload.

| `geometry_hash` | string | SHA-256 content hash computed from canonical feature geometry. |
| `properties_hash` | string | SHA-256 content hash computed from published non-geometry source properties. |
| `CC1` | string | Two-character GENC-derived political entity code for the first side of the boundary relationship. |
| `CC1_GENC3` | string | Three-character GENC code corresponding to `CC1`. |
| `CC1_WPID` | string | World Polygons UUID for the first side of the boundary relationship. |
| `COUNTRY1` | string | Department of State approved country or area name for the first side of the boundary relationship. |
| `CC2` | string | Two-character GENC-derived political entity code for the second side of the boundary relationship. |
| `CC2_GENC3` | string | Three-character GENC code corresponding to `CC2`. |
| `CC2_WPID` | string | World Polygons UUID for the second side of the boundary relationship. |
| `COUNTRY2` | string | Department of State approved country or area name for the second side of the boundary relationship. |
| `RANK` | string | Numeric expression of boundary status: `1` international boundary, `2` other line of international separation, `3` special line. |
| `LABEL` | string | Cartographic label text for line segments where scale permits. |
| `STATUS` | string | Boundary status text corresponding to `RANK`. |
| `NOTES` | string | Source notes explaining special circumstances, limitations, or original line source details. |
| `ID` | string | Source LSIB UUID for a specific feature instance; used as the generated feature ID assignment key. |
| `ANTECIDS` | string | Antecedent UUIDs for line geometries from which a given line is descended. |
| `PREVIDS` | string | Previous-version UUIDs for earlier versions of the conceptual line feature. |
| `PARENTID` | string | UUID linking related rank 2 or rank 3 lines to a corresponding rank 1 line. |
| `PARENTSEG` | string | Future-use field for constituent parts of boundary lines with the same rank or status. |
