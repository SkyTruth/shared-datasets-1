## What this is

This private asset is a geospatial conversion of ACLED event-level conflict and demonstration records for the Democratic Republic of Congo. It combines the supplied `ACLED Data DRC 2014-2025.csv` file with the rows from `ACLED_Dataset_Sub-SaharanAfrica 2026.xlsx` whose `country` value is `Democratic Republic of Congo`.

The source files share the same 31-column ACLED event schema. The release normalizes both files to that schema, de-duplicates on `event_id_cnty` with the newer Sub-Saharan Africa workbook taking precedence, filters to DRC, and writes WGS84 point geometry from the source `longitude` and `latitude` fields. The final release has 38,682 DRC event records from 2014-01-01 through 2026-06-29.

## When to use it

- Use this for private internal analysis of ACLED event-level conflict, disorder, actor, fatality, and source-observation patterns in the Democratic Republic of Congo.
- Use the `fgb` file as the analytical source and the `pmtiles` file for map preview or lightweight visual inspection.
- Use `event_id_cnty` or `feature_id` as the stable event lookup key for this release.
- Use `event_date`, `admin1`, `admin2`, `location`, `event_type`, `sub_event_type`, `actor1`, `actor2`, and `source` as primary search and filter fields.
- Do not use this as a public or redistributable dataset unless ACLED terms and SkyTruth authorization explicitly allow that use.
- Do not interpret the point geometry as exact event coordinates when ACLED `geo_precision` indicates approximate location coding.

## Schema notes

Geometry is WGS84 point geometry derived from ACLED `longitude` and `latitude`. The source coordinate fields are preserved as attributes. `geo_precision` and `time_precision` retain ACLED precision flags; consumers should account for those flags before treating an event as spatially or temporally exact.

During conversion, source field names were already lowercase snake case and were preserved. Four provenance fields were added: `source_file_name`, `source_url`, `source_accessed_date`, and `source_version`. The release feature model adds `feature_id`, `geometry_hash`, and `properties_hash`. `feature_id` is copied directly from ACLED `event_id_cnty` because it is unique, nonblank, and URL-safe for all 38,682 published rows.

The metadata sidecar contains one row per `feature_id` with full source properties and provenance. PMTiles features intentionally include only `feature_id` so clients resolve full source properties through the feature metadata sidecar. Firestore metadata serving is inactive for the first upload; the metadata sidecar is the durable release artifact.

No localized metadata sidecars were autogenerated for this first upload because no translation locales or metadata fields were requested.
