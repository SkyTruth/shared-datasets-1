## What this is

World Polygons is the U.S. Department of State Office of the Geographer and
Global Issues polygon dataset for countries and other State Department-recognized
areas. Version 11.4 beta was published on February 24, 2025. It is the polygon
companion to State Department boundary and geographic reference data, including
country and dependency names, GENC codes, display labels, recognition status, map
display hints, and source notes.

The source metadata marks this release as beta and describes it as provided for
testing and evaluation. The source guidance also says World Polygons is not a
standalone product for cartographic output: cartographic products should use it
with the State Department LSIB overlay and appropriate shoreline reference data.
The source package includes a Caspian Sea polygon as a data artifact; this
release preserves that row because it is present in the authoritative source, but
users should not treat it as a required cartographic water polygon.

## When to use it

- Use this as a reusable global State Department country and area polygon
  reference layer when U.S. Government-recognized names and political geography
  are the intended basis.
- Use the FlatGeobuf file for analytical workflows and metadata joins.
- Use the PMTiles file for web-map display and catalog preview.
- Pair this with LSIB and shoreline data for cartographic products that need the
  State Department boundary model.
- Do not use this as a neutral statement of de facto control, cadastral boundary
  source, shoreline engineering dataset, or substitute for specialist legal
  boundary analysis.
- Do not use the PMTiles artifact as the analytical source.

## Schema notes

Geometry is WGS84 multipolygon geometry. The source GeoPackage layer
`DoS_WP_11_4` reports 323 multipolygon features with extent `(-180, -90) -
(180, 83.666250)`. The published FlatGeobuf keeps the source fields plus
`feature_id`, `geometry_hash`, and `properties_hash`.

The source `WP_ID` field is unique and nonblank for 322 source rows, but it is
blank for the Caspian Sea artifact row. Since direct feature IDs must cover every
published feature, the reviewed 2026-06-26 release assigns generated monotonic
decimal `feature_id` values (`1`..`323`) from the stored `geometry_hash` and
`properties_hash` pair. The `WP_ID` value remains in the source attributes and
metadata sidecar for rows that have it.

The PMTiles artifact was built at zooms 0 through 12 from the release model as
metadata-lookup tiles. PMTiles feature properties are intentionally limited to
`feature_id`; full source attributes are available in the FlatGeobuf and metadata
sidecar.

No autogenerated localized metadata sidecars were requested for this first
upload.
